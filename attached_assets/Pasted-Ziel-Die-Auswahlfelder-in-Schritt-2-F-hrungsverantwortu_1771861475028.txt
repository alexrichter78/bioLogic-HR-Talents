Ziel

Die Auswahlfelder in Schritt 2 (Führungsverantwortung, Erfolgsfokus, Aufgabencharakter, Arbeitslogik) sollen sich nicht wie klassische Formularelemente anfühlen, sondern wie Apple-ähnliche Segmented / Pill Controls.

Kein Standard-Checkbox- oder Radio-Button-UI.

1. Grundprinzip: Pill Button Groups

Alle Auswahloptionen werden als Pill Buttons umgesetzt.

Technische Spezifikation

Button Height: 48px
Border Radius: 999px
Horizontal Padding: 28px
Font Size: 16px
Font Weight: 500
Line Height: 1

2. Visuelle Zustände
Inaktiv (Default)

Background: transparent
Text Color: #6E6E73
Border: none
Box Shadow: none

Hover:

Background: rgba(0, 0, 0, 0.04)

Cursor: pointer

Transition: 180ms ease

Aktiv (Selected)

Background: #0071E3
Text Color: #FFFFFF
Box Shadow: none
Border: none

Optional:

Subtle inner shadow: inset 0 0 0 1px rgba(0,0,0,0.04)

Transition:
transition: all 180ms cubic-bezier(0.4, 0, 0.2, 1);

3. Gruppierung

Buttons werden in einem gemeinsamen Container gerendert.

Container:

Display: inline-flex

Gap: 8px

Background: rgba(0,0,0,0.03)

Padding: 6px

Border Radius: 999px

Keine sichtbaren Borders zwischen Buttons.

4. Logik je Abschnitt
Führungsverantwortung

Single Selection (Radio-Logik)

Optionen:

Keine

Koordination

Führung

Nur ein Wert aktiv.

Erfolgsfokus

Multi Selection (max 2)

Optionen als Pill Buttons (nicht Checkboxen).

Beim dritten Klick:

Entweder älteste Auswahl deaktivieren

Oder visuelle Info: "Maximal 2 auswählbar"

Kein Alert-Popup.

Aufgabencharakter

Single Selection

Überwiegend operativ

Gemischt

Überwiegend strategisch

Arbeitslogik

Single Selection

Menschenorientiert

Daten-/prozessorientiert

Umsetzungsorientiert

5. Interaktion

Beim Klick auf Button:

Aktiver Zustand gleitet weich

Kein sofortiger harter Farbwechsel

Kein Springen

Keine Checkbox-Icons

Optional Micro-Animation:

transform: scale(1.02) on active

transition 120ms

7. Keine klassischen Formelemente

NICHT verwenden:

<input type="radio">

<input type="checkbox">

Default Browser Styles

Grau hinterlegte Formularfelder

Nur custom controlled components.

8. UX-Ziel

Die Auswahl soll sich anfühlen wie:

„Ich treffe eine Entscheidung.“

Nicht wie:

„Ich fülle ein Formular aus.“
