2. Die 13 Möglichkeiten
A. 6 klare Rangfolgen

Hier ist eine Komponente dominant, die zweite unterstützt, die dritte ist nachgeordnet.

IMP_INT_ANA

IMP_ANA_INT

INT_IMP_ANA

INT_ANA_IMP

ANA_IMP_INT

ANA_INT_IMP

B. 6 Doppeldominanzen

Hier sind zwei Komponenten nahezu gleich stark, die dritte liegt darunter.

IMP_INT_DUAL

IMP_ANA_DUAL

INT_IMP_DUAL ← technisch eigentlich identisch zu IMP_INT_DUAL, deshalb nicht doppelt anlegen

INT_ANA_DUAL

ANA_IMP_DUAL ← identisch zu IMP_ANA_DUAL, deshalb nicht doppelt anlegen

ANA_INT_DUAL ← identisch zu INT_ANA_DUAL, deshalb nicht doppelt anlegen

Für die tatsächliche Systemlogik reichen also diese 3 Doppeldominanz-Typen nicht aus, wenn du die Reihenfolge zur dritten Komponente unterscheiden willst.
Da du aber von 13 Möglichkeiten sprichst, ist die übliche bioLogic-Logik:

6 eindeutige Reihenfolgen

6 Doppeldominanzen mit Richtung zur dritten Komponente

1 Balance

Also technisch:

IMP_INT__ANA

IMP_ANA__INT

INT_IMP__ANA

INT_ANA__IMP

ANA_IMP__INT

ANA_INT__IMP

Diese sechs Keys beschreiben:

welche zwei vorne gleich stark sind

welche Komponente hinten liegt

C. 1 Balance-Profil

BALANCED

Das bedeutet:

alle drei Werte liegen in einem definierten Toleranzbereich

keine klare Dominanz

3. Empfohlene Logik zur Profil-Erkennung
3.1 Eingangsdaten
type DimensionName = "impulsiv" | "intuitiv" | "analytisch";

type RoleProfileInput = {
  impulsiv: number;
  intuitiv: number;
  analytisch: number;
  isLeadershipRole: boolean;
};
3.2 Konfigurierbare Schwellen
const DUAL_THRESHOLD = 5;       // max. Differenz für Doppeldominanz
const BALANCED_THRESHOLD = 5;   // max. Differenz zwischen höchstem und niedrigstem Wert
3.3 Entscheidungslogik
Schritt 1: Balance prüfen

Wenn gilt:

maxValue - minValue <= BALANCED_THRESHOLD

dann:

resultKey = "BALANCED"
dominanceType = "balanced"
Schritt 2: Doppeldominanz prüfen

Wenn die Differenz zwischen Platz 1 und Platz 2 kleiner oder gleich DUAL_THRESHOLD ist,
aber Platz 3 klar darunter liegt, dann liegt eine Doppeldominanz vor.

Beispiel:

Impulsiv: 42

Intuitiv: 40

Analytisch: 18

Dann:

resultKey = "IMP_INT__ANA"
dominanceType = "dual"
Schritt 3: Klare Reihenfolge

Wenn weder Balance noch Doppeldominanz vorliegt, dann wird nach Rangfolge sortiert.

Beispiel:

Analytisch 51

Intuitiv 31

Impulsiv 18

Dann:

resultKey = "ANA_INT_IMP"
dominanceType = "single"

5. Wichtig für den Programmierer

Ob du bei Doppeldominanzen wirklich IMP_INT__ANA und INT_IMP__ANA getrennt brauchst, hängt davon ab, wie dein Berechnungsmodell intern priorisiert.

Empfehlung

Wenn beide ersten Werte im Doppelfall wirklich gleichwertig behandelt werden, reicht technisch:

IMP_INT__ANA

IMP_ANA__INT

INT_ANA__IMP

Dann wären es aber nicht 13, sondern 10 Typen.

Da du aber ausdrücklich mit 13 Möglichkeiten arbeitest, sollte der Programmierer alle 13 Keys vollständig abbilden, auch wenn einzelne Doppelformen textlich später zusammengeführt werden.

15. Beispiel für Berechnungsfunktion
type RankedDimension = {
  key: "IMP" | "INT" | "ANA";
  value: number;
};

function getRoleResultKey(
  impulsiv: number,
  intuitiv: number,
  analytisch: number
): ResultKey {
  const values: RankedDimension[] = [
    { key: "IMP", value: impulsiv },
    { key: "INT", value: intuitiv },
    { key: "ANA", value: analytisch }
  ];

  const sorted = [...values].sort((a, b) => b.value - a.value);

  const max = sorted[0].value;
  const mid = sorted[1].value;
  const min = sorted[2].value;

  if (max - min <= BALANCED_THRESHOLD) {
    return "BALANCED";
  }

  if (max - mid <= DUAL_THRESHOLD && mid - min > DUAL_THRESHOLD) {
    return `${sorted[0].key}_${sorted[1].key}__${sorted[2].key}` as ResultKey;
  }

  return `${sorted[0].key}_${sorted[1].key}_${sorted[2].key}` as ResultKey;
}
