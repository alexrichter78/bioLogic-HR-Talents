Nachgedacht für 19s

Ja. Das ist genau der richtige Ansatz: Die fachliche Bewertung bleibt unverändert, aber die Darstellung bekommt eine feinere visuelle Abstufung.

Der Denkfehler beim Kunden entsteht, weil „bedingt geeignet“ optisch wie eine harte Mittelstufe wirkt. In Wirklichkeit kann „bedingt geeignet“ aber zwei völlig verschiedene Dinge bedeuten:

bedingt geeignet, aber fast geeignet
Beispiel: minimale Strukturabweichung, praktisch sehr nahe dran.
bedingt geeignet, aber schon kritisch
Beispiel: mehrere deutliche Abweichungen, noch nicht „nicht geeignet“, aber mit spürbarem Entwicklungsaufwand.

Deshalb würde ich die 3 Balken nicht ersetzen, sondern ergänzen durch eine 12-Punkte-Feinanzeige.

Lösung: 12-Punkte-Skala „Passungsnähe“

Warum kann trotzdem „bedingt geeignet“ entstehen?

Weil bei deiner Logik nicht nur die Prozentabweichung zählt, sondern auch die Strukturrelation.

Beim Soll liegen Impulsiv und Analytisch mit 36 zu 41 noch innerhalb der 5-Punkte-Toleranz. Also strukturell fast gleich.

Beim Ist liegt Analytisch mit 41 zu 35 aber 6 Punkte über Impulsiv. Damit kippt eine Relation knapp aus der Gleichheit heraus.

Das ist fachlich sauber als Soft Conflict bewertbar. Aber optisch muss sichtbar werden:

Das ist kein ernstes Problem. Das ist ein Grenzfall direkt neben „geeignet“.

Also wäre die Anzeige:

Geeignet        Bedingt geeignet        Nicht geeignet
● ● ● ●   |   ◉ ○ ○ ○   |   ○ ○ ○ ○

Oder als Zahl:

Passungsnähe: 5 von 12

Das heißt:

Bedingt geeignet, aber direkt an der Grenze zu geeignet.

Einfache Formel für deine Software

Du kannst eine zusätzliche Variable bauen:

visualFitPoint

Diese Variable verändert nicht das Ergebnis, sondern nur die Anzeige.

Beispiel-Logik:

function getVisualFitPoint({ fitLabel, structureResult, maxDiff, totalGap }) {
  // Geeignet
  if (fitLabel === "Geeignet") {
    if (maxDiff <= 1 && totalGap <= 3) return 1;
    if (maxDiff <= 3) return 2;
    if (maxDiff <= 4) return 3;
    return 4;
  }

  // Bedingt geeignet
  if (fitLabel === "Bedingt geeignet") {
    // Soft Conflict mit sehr kleinen Abweichungen:
    // fachlich bedingt, optisch nahe an geeignet
    if (structureResult === "SOFT_CONFLICT") {
      if (maxDiff <= 2) return 5;
      if (maxDiff <= 5) return 6;
      if (maxDiff <= 8) return 7;
      return 8;
    }

    // Exact, aber numerisch über Geeignet-Grenze
    if (structureResult === "EXACT") {
      if (maxDiff <= 6) return 5;
      if (maxDiff <= 7) return 6;
      if (maxDiff <= 9) return 7;
      return 8;
    }

    // Fallback für andere bedingte Fälle
    if (maxDiff <= 4) return 5;
    if (maxDiff <= 7) return 6;
    if (maxDiff <= 10) return 7;
    return 8;
  }

  // Nicht geeignet
  if (fitLabel === "Nicht geeignet") {
    if (maxDiff <= 12) return 9;
    if (maxDiff <= 18) return 10;
    if (maxDiff <= 25) return 11;
    return 12;
  }

  return 8;
}
