1. Struktur bleibt immer objektiv
1.1 MatchCase nur aus realer Struktur ableiten

matchCase darf nie aus dem BALANCED-Fall „vergeben“ werden.

type MatchCase = "TOP1_TOP2" | "TOP1_ONLY" | "TOP2_ONLY" | "NONE";

function getMatchCase(team: ProfileMeta, person: ProfileMeta): MatchCase {
  if (team.top1 === person.top1 && team.top2 === person.top2) return "TOP1_TOP2";
  if (team.top1 === person.top1) return "TOP1_ONLY";
  if (team.top2 === person.top2) return "TOP2_ONLY";
  return "NONE";
}

Das ist dann ein reiner Struktur-Fakt.

1.2 Neuer Sonderfall: integrationCase

Für BALANCED und andere Sonderkonstellationen führst du zusätzlich ein zweites Feld ein:

type IntegrationCase =
  | "STRUCTURE_STRONG"
  | "STRUCTURE_MEDIUM"
  | "STRUCTURE_WEAK"
  | "BALANCED_BALANCED"
  | "TEAM_BALANCED_OPEN"
  | "TEAM_BALANCED_SELECTIVE"
  | "TEAM_BALANCED_TENSE"
  | "PERSON_BALANCED_ADAPTIVE"
  | "PERSON_BALANCED_LIMITED";

Damit wird nicht mehr der MatchCase verbogen.

2. Profilklassifikation bleibt, aber BALANCED wird sauber integriert

Deine Klassifikation ist grundsätzlich gut:

BALANCED: gap1 <= 5 && gap2 <= 5
DUAL:     gap1 <= 5 && gap2 > 5
CLEAR:    gap1 > 5  && gap2 <= 5
ORDER:    gap1 > 5  && gap2 > 5

Die würde ich so lassen.

function getSpreadClass(spread: number): "balanced" | "eng" | "nah" | "fern" {
  if (spread <= 5) return "balanced";

Konstellation	Score	integrationCase
beide BALANCED	95	BALANCED_BALANCED

Das ist wesentlich sauberer als die alte Matrix.

3.5 Implementation
function computeBalancedScore(team: ProfileMeta, person: ProfileMeta) {
  const teamBalanced = team.profileClass === "BALANCED";
  const personBalanced = person.profileClass === "BALANCED";

  if (!teamBalanced && !personBalanced) return null;

  const teamSpreadClass = getSpreadClass(team.spread);
  const personSpreadClass = getSpreadClass(person.spread);

  if (teamBalanced && personBalanced) {
    return { score: 95, integrationCase: "BALANCED_BALANCED", scoreMode: "balanced_override" as const };
  }

  if (teamBalanced) {
    if (personSpreadClass === "eng") {
      return { score: 82, integrationCase: "TEAM_BALANCED_OPEN", scoreMode: "balanced_override" as const };
    }
    if (personSpreadClass === "nah") {
      return { score: 72, integrationCase: "TEAM_BALANCED_SELECTIVE", scoreMode: "balanced_override" as const };
    }
    return { score: 60, integrationCase: "TEAM_BALANCED_TENSE", scoreMode: "balanced_override" as const };
  }

  if (personBalanced) {
    if (teamSpreadClass === "eng") {
      return { score: 80, integrationCase: "PERSON_BALANCED_ADAPTIVE", scoreMode: "balanced_override" as const };
    }
    if (teamSpreadClass === "nah") {
      return { score: 72, integrationCase: "PERSON_BALANCED_ADAPTIVE", scoreMode: "balanced_override" as const };
    }
    return { score: 60, integrationCase: "PERSON_BALANCED_LIMITED", scoreMode: "balanced_override" as const };
  }

  return null;
}

7. Systemwirkung neu ableiten

Das war in V4 zu grob. In V4.1 sollte systemwirkung nicht nur an matchCase hängen.

7.1 Neue Logik
type Systemwirkung =
  | "Verstärkung"
  | "Stabile Ergänzung"
  | "Ergänzung mit Spannung"
  | "Selektive Ergänzung"
  | "Anpassungsfähige Ergänzung"
  | "Transformation";
7.2 Ableitung
function computeSystemwirkung(params: {
  matchCase: MatchCase;
  integrationCase?: IntegrationCase;
  passungZumTeam: "hoch" | "mittel" | "gering";
  team: ProfileMeta;
  person: ProfileMeta;
}): Systemwirkung {
  const { matchCase, integrationCase, passungZumTeam, team, person } = params;

  if (integrationCase === "BALANCED_BALANCED") return "Stabile Ergänzung";
  if (integrationCase === "TEAM_BALANCED_OPEN") return "Stabile Ergänzung";
  if (integrationCase === "TEAM_BALANCED_SELECTIVE") return "Selektive Ergänzung";
  if (integrationCase === "TEAM_BALANCED_TENSE") return "Ergänzung mit Spannung";
  if (integrationCase === "PERSON_BALANCED_ADAPTIVE") return "Anpassungsfähige Ergänzung";
  if (integrationCase === "PERSON_BALANCED_LIMITED") return "Ergänzung mit Spannung";

  if (matchCase === "TOP1_TOP2") return "Verstärkung";
  if (matchCase === "TOP1_ONLY") return "Stabile Ergänzung";
  if (matchCase === "TOP2_ONLY") return "Ergänzung mit Spannung";

  if (matchCase === "NONE" && passungZumTeam === "mittel") return "Selektive Ergänzung";
  return "Transformation";
}

Warum ist das besser?

NONE ist nicht mehr automatisch immer Transformation
BALANCED bekommt eine eigene Wirkung
Text und Score laufen weniger auseinander
8. Varianten-Kompatibilität leicht schärfen

Deine bisherige Kompatibilität ist okay, aber etwas grob.

Bisher:

DUAL ↔ ORDER = kompatibel
CLEAR ↔ ORDER = kompatibel

Mir fehlt:

DUAL ↔ CLEAR ist je nach Struktur teils auch näher als gar nichts

Ich würde das so machen:

function getVariantCompatibility(a: ProfileMeta, b: ProfileMeta): number {
  if (a.variant === b.variant) return 10;
  if (a.profileClass === b.profileClass) return 7;

  const pair = `${a.profileClass}_${b.profileClass}`;

  if (
    pair === "DUAL_ORDER" || pair === "ORDER_DUAL" ||
    pair === "CLEAR_ORDER" || pair === "ORDER_CLEAR" ||
    pair === "DUAL_CLEAR" || pair === "CLEAR_DUAL"
  ) {
    return 5;
  }

  return 0;
}

Das ist etwas runder.
