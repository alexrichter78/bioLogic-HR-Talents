Die saubere Lösung

Die Struktur darf nicht nur über Variantennamen geprüft werden.
Sie muss über die 3 Paarvergleiche geprüft werden:

I vs N
I vs A
N vs A

Für jeden Vergleich gilt:

0 = gleich stark, wenn Differenz <= 5
1 = erste Komponente klar stärker
-1 = zweite Komponente klar stärker

Dadurch bekommt jedes Profil intern eine Struktur aus genau 3 Relationen.

Beispiel
I = N > A

wird zu:

I vs N = 0
I vs A = 1
N vs A = 1
I > N > A

wird zu:

I vs N = 1
I vs A = 1
N vs A = 1

Der Unterschied ist also nur eine einzige Relation:
I vs N wechselt von 0 auf 1

Das ist ein echter weicher Grenzfall.

Schritt 3: Strukturvergleich über Paarrelationen

Für SOLL und IST werden diese 3 Relationen gebildet:

(I,N)
(I,A)
(N,A)

Werte jeweils:

0 = gleich (<= 5)
1 = erste klar stärker
-1 = zweite klar stärker
Schritt 4: Strukturbeziehung bestimmen
Exakt

Alle 3 Relationen sind identisch.

→ exakte Struktur

Weicher Strukturkonflikt

Genau 1 der 3 Relationen ist unterschiedlich
und es ist kein direkter Flip von 1 zu -1 oder umgekehrt.

Also nur:

0 ↔ 1
oder 0 ↔ -1

→ weicher Strukturkonflikt

Harter Strukturkonflikt

Sobald:

mehr als 1 Relation abweicht
oder irgendwo ein direkter Flip 1 ↔ -1 vorkommt

→ harter Strukturkonflikt

Damit sind alle 13 Varianten sauber abgedeckt.

Damit werden die früheren Fehler vermieden:

Früher fälschlich zu weich
ALL_EQUAL ↔ BOTTOM_PAIR

Das ist nicht weich, weil hier 2 Relationen kippen, nicht nur 1.

Früher fälschlich zu hart
I = N > A gegen N > I > A

Das ist nur ein Wechsel von I:N = 0 zu I:N = -1
also weich, nicht hart.
